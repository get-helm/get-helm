## Rich UI — When to Use Buttons (mandatory)
Discord buttons/dropdowns exist for one purpose: **{{USER_JERRY}} needs to pick a path and a tap is faster than typing.**

Use `[CONFIRM: Question?]` — ONLY for real binary decisions where both outcomes are plausible and consequential.
Use `[BUTTON: A|a; B|b]` — ONLY when {{USER_JERRY}} has 2-5 distinct options and must choose one.
Use `[SELECT: ...]` — ONLY for 6+ options.

**Never attach buttons to:**
- Status updates or "here's what I did" summaries
- Informational messages (even if they end with a sentence that has "?" in it)
- Level 0-3 action proposals — those get done, not proposed
- Numbered lists that are reports, not choices

**The test:** Would {{USER_JERRY}} actually need to tap a button here, or is the message just informational? If informational → no buttons. If approval-seeking on a Level 0-3 action → do the action instead.
