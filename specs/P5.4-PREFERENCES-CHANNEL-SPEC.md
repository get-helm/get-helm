# P5.4: User Preferences Management — Complete Specification

**Status:** Design (awaiting approval)  
**Effort:** 2 hours  
**Option:** A evolved (#preferences channel with interactive updates)  

---

## Overview

Instead of a web form, user manages settings via `#preferences` Discord channel. Pinned message shows all current settings. User updates by asking `@HELM set [setting] to [value]` in any channel (or specifically in #preferences).

**User experience:** Settings are visible, discoverable, and changeable without leaving Discord.

---

## Component 1: #preferences Channel

**Location:** Auto-created in every HELM instance (by scaffolder)  
**Members:** The HELM user(s) only  
**Purpose:** Central hub for preference management  

**Channel description:**
```
Manage your HELM preferences here. See what's currently set, change anything anytime.
Examples: @HELM set pushback_volume to "low", @HELM change my communication style to professional
```

## Component 3: Setting Change Command

**How it works:**

User (any channel): `@HELM set pushback_volume to low`

Bot:
1. Parses the setting name ("pushback_volume")
2. Validates the new value ("low" is a valid option)
3. Updates `settings.json`
4. Confirms in the channel
5. Updates the pinned #preferences message

**Bot response:**

```
✅ Updated: pushback_volume
  old: Often (challenge my ideas frequently)
  new: Low (challenge occasionally)

Updated pinned preferences → see #preferences
```

**If invalid:**

```
❌ Can't set "pushback_volume" to "extremely-low"

Valid options: often, occasionally, never

Try: @HELM set pushback_volume to occasionally
```

## Component 6: Update Flow

**Sequence when user changes a setting:**

```
1. User: @HELM set pushback_volume to low
2. Bot parses: setting="pushback_volume", value="low"
3. Bot validates: "low" is in [often, occasionally, never]
4. Bot updates settings.json
5. Bot posts confirmation: "✅ Updated: pushback_volume"
6. Bot fetches full settings.json
7. Bot regenerates pinned message in #preferences
8. Bot calls discord-update-pinned-message.sh
9. Pinned message updated in real-time
```

## Implementation Details

**Code location:** New `preferences-command.md` agent OR extend `help.md`  
**Trigger:** Message contains `@HELM set [...]` OR `@HELM change [...]`  
**Storage:** settings.json (already exists from P5.1 onboarding)  
**UI update:** discord-update-pinned-message.sh (utility script)  

**Steps:**
1. Parse message for setting name + value (regex or NLP)
2. Look up in settings-registry.md to validate + describe
3. If valid: update settings.json + confirm + refresh pinned message
4. If invalid: suggest valid values + link to `@HELM help preferences`

## Success Criteria

1. ✅ User sees all current settings in #preferences (pinned)
2. ✅ User can change any setting via `@HELM set [...]` in any channel
3. ✅ Pinned message updates in real-time (within 2 sec)
4. ✅ Invalid setting values are rejected with helpful error message
5. ✅ Settings persist across bot restarts
6. ✅ No web form needed — all in Discord

---

## Why This Approach

**Simpler than a web form:**
- No new URL, no new auth, no new infrastructure
- Everything stays in Discord where the user is
- Settings are discoverable (pinned message)

**More discoverable than CLI only:**
- User sees all options (not remembering command syntax)
- Examples are pinned right there
- Can browse settings before changing

**Fits HELM's philosophy:**
- Discord-native
- No external tools
- User never leaves the channel
