# P5.1: HELM Onboarding Flow — Complete Specification

**Status:** Design (awaiting approval)
**Rewrite:** 2026-06-06 — full rebuild from scratch per session feedback
**Source:** pap-onboarding-script.md (canonical questions), session feedback, web research

---

## Design Principles

Every step passes all of these. Non-negotiable.

1. **Assume they know nothing** — the user's only prior knowledge is "my friend uses this." Write for that person at every step.
2. **Race to the aha moment** — first time the AI responds to them personally in Discord. Every step before that is overhead; minimize it.
3. **Do the work for them** — HELM navigates, opens URLs, pre-selects options. The user approves. They do not execute.
4. **One thing at a time** — never show two decisions at once. One question. One action. Wait.
5. **Context before the ask** — before requesting anything, explain why in one sentence. Benefit-framed.
6. **Celebrate every win** — non-technical users need confidence. Confirm every step visibly.
7. **Safe exits everywhere** — "I'm stuck" or "skip for now" on every step. Forced flows have higher abandonment.
8. **Default everything possible** — every question is an exit risk. If a default is almost always right, use it.
9. **Email after first value** — never ask for personal information before the user has seen HELM do something useful.
10. **Collect preferences over time** — 3 taps at first. More preferences on days 3 and 7, not upfront.
11. **Friendly, not chatty** — warm and direct. One human might say it to another. Never corporate, never mechanical.
12. **Design for "my friend recommended this"** — the user's first impression is "my friend said this helps them." Lean into that. Make them feel like they're joining something.

## Flow at a Glance

```
PHASE 0: GitHub Landing Page
    (user copies the pre-install prompt)
PHASE 1: Pre-Install Guide   [daily machine, Claude.ai]
    (wipe + set up clean machine, install Claude Desktop, get subscription)
PHASE 2: HELM Installation   [clean machine, Claude Desktop Code tab → Local mode]
    (HELM installs, Discord bot created, first Discord message sent)
PHASE 3: Getting Started     [Discord]
    (3 taps at first, more on day 3 and 7)
HELM is live
```

**What each interface is responsible for:**

| Interface | What happens here | Why here |
|---|---|---|
| GitHub page | Convince user to start. One action only. | No auth, no friction, zero-commitment. |
| Claude.ai (daily machine) | Hardware check, wipe guide, get Claude sub, install Claude Desktop | User is still on old machine. AI is patient, interactive, handles confusion. |
| Claude Desktop Code tab → Local (clean machine) | Full HELM install, Discord bot creation, first setup | Local mode has filesystem access to actually install HELM. |
| Discord | All preference questions, stages 1-3, everything after | HELM lives here. Natural habitat. |
