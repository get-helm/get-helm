## Component 3: Interactive `@HELM help [topic]` Command

**How it works:**

User: `@HELM help phases`

Bot:
1. Parses the topic ("phases")
2. Searches GitHub docs for matching section (GUIDE.md "## Phases")
3. Extracts 3-5 relevant paragraphs (500 tokens max)
4. Posts with link to full docs

Bot response example:

```
📖 Phases (from HELM docs)

HELM breaks every automation into four phases:

**Phase A: Understand**
You describe the problem, HELM asks questions, then designs the solution.
Effort: 15–30 min. You decide yes/no.

**Phase B: Validate** 
HELM builds a rough version. You test it, it improves iteratively.
Effort: 2–8 hours (depends on automation).

**Phase C: Optimize**
Once it works, HELM refines it (speed, accuracy, reliability).
Effort: 2–4 hours.

**Phase D: Graduate**
The automation runs on its own. HELM maintains it if needed.
Effort: <30 min (usually just approval).

Full guide: https://github.com/{{USER_GITHUB}}13/helm-docs/blob/main/GUIDE.md#phases
```

**Indexing:**
- Bot pre-indexes GitHub docs (headings + subsections)
- Topics are: "phases", "proposals", "workspaces", "preferences", "commands", "troubleshooting", "faq", "what is HELM", "phases", "phase-a", "phase-b", etc.
- Fuzzy matching ("phase" → "phases", "help" → returns FAQ)

**Implementation:**
```python
# Pseudo-code
def helm_help(topic):
    index = load_docs_index()  # pre-built from GitHub
    match = find_best_match(topic, index)  # fuzzy search
    if match:
        section = fetch_from_github(match['url'])
        excerpt = extract_excerpt(section, max_tokens=500)
        return format_response(excerpt, match['full_url'])
    else:
        return "Topic not found. Try: " + suggest_similar(topic)
```
